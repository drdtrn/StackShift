**Top 3 lessons across the project:**
1. **Groundedness beats generality** — feeding the AI both the requirements doc AND the current codebase state in one prompt produces plans grounded in reality; it spots missing packages, stale stubs, and gaps without being told.
2. **Sealed types and scoping are the recurring ambush** — sealed SDK types (OpenAI `ChatClient`, EF InMemory) and HTTP-context-scoped services (org repos) silently fail in background job scope; discovering this from code rather than docs saves hours of debugging.
3. **Prompt iteration is non-negotiable for novel integrations** — BE-10 (AI RAG) required 5 distinct prompts before the architecture was correct; front-loading the iteration mid-week (not the day before the deadline) is what made it shippable.

> **Honest failure documented:** BE-10 Prompt v2 injected `ChatClient` directly — Moq threw `Can't create a Mock of a sealed type`; required a full redesign to `IChatCompleter`/`IEmbedder` thin wrappers before the test layer could be built. BE-20 (⭐⭐⭐⭐ Required Edits) shows AI hallucinating domain enum values — domain enums must always be pasted verbatim into the prompt.
